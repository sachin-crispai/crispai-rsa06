# Cluster 12 — API Security

**Stack:** Discover, monitor, and protect APIs — the most exposed attack surface in modern apps.

APIs now carry more sensitive data than traditional web apps and are the primary target for data breaches. API security is one of the fastest-growing RSAC categories.

## Technology Landscape

### API Discovery & Inventory
Find all APIs — including shadow/rogue APIs developers forgot about.
- **Leaders:** Salt Security, Noname, Traceable, Akamai, Imperva

### API Threat Protection (Runtime)
Detect and block API abuse — credential stuffing, BOLA, data scraping, injection.
- **Leaders:** Salt Security, Cequence, Akamai, Traceable

### API Posture Management
Assess API configuration, authentication, encryption, and compliance gaps.
- **Leaders:** Noname, 42Crunch, Salt Security

### API Testing (Shift-Left)
Test API contracts and security in CI/CD pipelines.
- **Leaders:** StackHawk, 42Crunch, Postman Security, OWASP API Security Top 10 tooling

---

## Key Questions to Ask at Booths
- How do you handle API discovery in microservices environments with 1000s of APIs?
- OWASP API Security Top 10 coverage — which categories do you detect?
- How do you differentiate legitimate automated API clients from malicious bots?
- LLM/AI agent APIs — do you support securing model inference endpoints?
- Time to first value: how fast can you get full API inventory in a new environment?
