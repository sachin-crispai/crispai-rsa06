# Cluster 15 — OT / ICS / IoT Security

**Stack:** Protect critical infrastructure, industrial control systems, and connected devices — where a breach can be physical.

OT/ICS security requires passive, agentless approaches that don't disrupt production. IoT security spans healthcare, manufacturing, smart buildings, and consumer devices.

## Technology Landscape

### OT/ICS Visibility & Monitoring
Passive monitoring of industrial protocols (Modbus, DNP3, EtherNet/IP) to detect anomalies.
- **Leaders:** Dragos, Claroty, Nozomi Networks, Microsoft Defender for IoT

### IoT/XIoT Security
Discover and secure the full spectrum: IoT, OT, ICS, BAS, medical devices.
- **Leaders:** Armis, Claroty, Forescout, Axonius

### Medical Device Security (HDO)
Healthcare delivery organization-specific: FDA compliance, clinical risk, device patching.
- **Leaders:** Cylera, Medigate (Claroty), Ordr

### OT Vulnerability Management
CVE tracking for ICS components — PLCs, HMIs, RTUs — with patch guidance.
- **Leaders:** Tenable OT, Dragos, Claroty

### OT Network Segmentation
Isolate OT networks from IT without disrupting production.
- **Leaders:** Elisity, Illumio, Fortinet (FortiGate NGFW in OT DMZ)

## Key Questions to Ask at Booths
- How do you handle proprietary industrial protocols without dedicated decoders?
- Active vs. passive scanning: what's the risk to production systems?
- How do you integrate OT visibility into existing IT SIEM workflows?
- Purdue Model vs. ISA-95: which architecture do you support for segmentation?
- Medical devices: FDA cybersecurity guidance 2023 compliance support?
